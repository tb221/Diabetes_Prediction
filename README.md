# Diabetes Prediction using Machine Learning

This repository contains two Jupyter notebooks that demonstrate the complete workflow of building a Diabetes Prediction system using Machine Learning with Scikit-learn. The project covers everything from data analysis and preprocessing to model training, evaluation, and deployment-ready inference using saved models.

---

# 1. diabetes.ipynb

## Overview

This notebook focuses on the complete Machine Learning workflow for predicting diabetes. It includes exploratory data analysis (EDA), data preprocessing, feature engineering, model comparison, and evaluation of multiple machine learning algorithms.

## Features

- Load and explore the Diabetes dataset
- Perform Exploratory Data Analysis (EDA)
- Analyze numerical and categorical features
- Build preprocessing pipelines using:
  - StandardScaler
  - OneHotEncoder
  - ColumnTransformer
- Train and compare multiple Machine Learning models
- Evaluate models using Cross Validation
- Compare model performance based on accuracy
- Select the best-performing classifier

## Machine Learning Algorithms Used

- Logistic Regression
- Random Forest Classifier
- Decision Tree Classifier
- Support Vector Machine (SVM)
- K-Nearest Neighbors (KNN)
- Naive Bayes
- SGD Classifier
- Extra Trees Classifier
- Gradient Boosting Classifier
- AdaBoost Classifier
- Passive Aggressive Classifier
- Linear Discriminant Analysis (LDA)
- Quadratic Discriminant Analysis (QDA)

## Libraries Used

- NumPy
- Pandas
- Matplotlib
- Seaborn
- Scikit-learn

# 2. final_diabetes.ipynb

## Overview

This notebook converts the machine learning model into a deployment-ready prediction pipeline. It trains the selected model, saves both the preprocessing pipeline and trained classifier using Joblib, and performs inference on unseen data.

## Features

- Stratified Train-Test Split
- Separate preprocessing pipelines for:
  - Numerical Features
  - Categorical Features
- Feature scaling using StandardScaler
- Categorical encoding using OneHotEncoder
- Combined preprocessing using ColumnTransformer
- Train a Random Forest Classifier
- Save the trained model using Joblib
- Save the preprocessing pipeline separately
- Load saved model and pipeline
- Perform inference on new input data
- Export prediction results to a CSV file

## Workflow

1. Load the Diabetes dataset.
2. Split the dataset using Stratified Shuffle Split.
3. Create preprocessing pipelines for numerical and categorical features.
4. Transform the training data.
5. Train the Random Forest model.
6. Save the trained model (`model.pkl`).
7. Save the preprocessing pipeline (`pipeline.pkl`).
8. Load both files for inference.
9. Predict diabetes for unseen data.
10. Save predictions to `output.csv`.

# Project Files

| File | Description |
|------|-------------|
| `diabetes.ipynb` | Performs Exploratory Data Analysis (EDA), data preprocessing, model training, cross-validation, and comparison of multiple machine learning algorithms. |

# Testing the Model

The project demonstrates how a trained machine learning model can be used for inference on unseen data.

- `input.csv` contains the feature values **without** the target (`Diabetes`) column.
- The trained preprocessing pipeline (`pipeline.pkl`) transforms the input features.
- The trained model (`model.pkl`) predicts whether each patient has diabetes.
- The predictions are saved in `output.csv`.
- To verify the model's predictions, compare `output.csv` with `diabetes_test.csv`, which contains the actual target labels for the same records.

This separation of input features and ground-truth labels simulates a real-world deployment scenario, where the model receives only feature values and generates predictions that can later be evaluated against the true outcomes.
